# Git Discipline

---

## Commit Principles

**One logical change per commit.** A commit that "fixes the flywheel and also cleans up imports" 
is two commits. Split them.

**Commits must compile.** Never commit code that does not pass `./gradlew compileJava`.

**Commit messages describe the why, not the what:**
```
GOOD: "Increase feeder alignment timeout to handle slow auto-aim at far distances"
BAD:  "Changed alignmentTimeoutSeconds from 0.9 to 1.1"

GOOD: "Fix vision rejecting good single-tag poses near field boundary"  
BAD:  "Updated vision filter"
```

---

## Branch Strategy

Work on a feature branch. Never push directly to `main` while the robot is at competition.

Branch naming: `feature/description`, `fix/description`, `tune/subsystem-name`

---

## What to Stage (Never Stage These)

```
NEVER stage:
  *.hoot files (match logs — too large, private)
  /U/ directory contents
  .env files
  Any file with credentials, IPs, or passwords
```

Stage only `.java`, `build.gradle`, `.json` vendordep files, `.auto` PathPlanner files, 
and configuration files that belong in version control.

---

## Pre-Commit Checklist

Before every commit:
- [ ] `./gradlew compileJava` passes
- [ ] `./gradlew spotlessCheck` passes (or `spotlessApply` was run)
- [ ] No unintended files in `git diff --staged`
- [ ] No debug `System.out.println()` left in code (use `Logger.recordOutput()`)
- [ ] No `TODO: delete this` or `TEMP:` comments left in

---

## Reverting at Competition

If a change causes a problem at competition and needs to be reverted:
```bash
git revert HEAD         # safe revert, creates new commit
# NOT: git reset --hard  (destroys history, dangerous on shared branch)
```

For anything more complex than a single-commit revert, **ask before acting**.

---

## Auto-Generated Files

`BuildConstants.java` is auto-generated at deploy time with git metadata.
Do not edit it manually. Do not add it to `.gitignore` — it should be committed 
after each deploy so match logs can be matched to source.

`generated/TunerConstants.java` and `generated/COMP_TunerConstants.java` — 
these ARE intentionally committed. They are generated by CTRE Tuner X but 
treated as source of truth for swerve configuration.
